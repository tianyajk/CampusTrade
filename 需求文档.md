# CampusTrade 校园二手交易平台 — 需求文档

---

## 一、项目内容

### 1.1 项目名称
CampusTrade（校园二手交易平台）

### 1.2 项目背景
大学校园内存在大量闲置物品（教材、电子产品、生活用品等），学生之间缺乏便捷、可信的二手交易渠道。学生通常只能通过微信群、QQ群或线下集市的方式交易，效率低下且缺乏规范管理。

### 1.3 项目目标
开发一个面向在校大学生的校园二手交易系统，提供从商品发布、浏览搜索、收藏、在线沟通到下单购买的一站式交易服务，让学生闲置物品流转更高效、更安全。

### 1.4 项目定位
- 课程项目，不要求高并发，不需要复杂分布式架构
- 核心关注：代码规范、项目结构清晰、可稳定运行
- 目标用户：在校大学生（买家 / 卖家）

### 1.5 技术选型

| 层级 | 技术 |
|------|------|
| 开发语言 | C# |
| 框架 | ASP.NET Core Web API |
| 数据库 | MySQL |
| ORM | Entity Framework Core |
| 前端 | HTML + CSS + Bootstrap + 原生 JavaScript（fetch） |
| 认证方式 | ASP.NET Core Cookie 认证（浏览器自动携带） |
| 架构 | 前后端分离（单项目，API 控制器 + 静态前端） |
| 开发工具 | VS Code |

### 1.6 项目结构

```
CampusTrade/
├── backend/
│   ├── Controllers/        # API 控制器（返回 JSON）
│   ├── Models/             # 数据库实体 + DTO（请求/响应模型）
│   ├── Services/           # 业务逻辑 + 数据访问（直接注入 DbContext）
│   ├── Data/               # DbContext、数据库迁移
│   └── Program.cs          # 应用入口、服务注册、静态文件指向 frontend/
│
├── frontend/
│   ├── pages/              # HTML 页面
│   ├── js/                 # JavaScript（API 调用、页面交互）
│   ├── app.css             # 自定义样式（Bootstrap 之外的补充）
│   └── assets/             # 图片、图标
│
└── CampusTrade.csproj
```

**说明**：
- 后端纯 API，只返回 JSON，无 Razor 视图
- Models/ 同时放实体类和 DTO，不单独拆目录
- 前端纯静态文件，`fetch` 调用后端 API，无需 Node.js
- `UseStaticFiles` 直接指向 `frontend/` 目录，不设 wwwroot

### 1.7 开发规范

| 规范项 | 要求 |
|--------|------|
| 命名规范 | 类名 PascalCase，变量名 camelCase |
| 注释 | 每个方法添加注释说明 |
| 依赖注入 | 统一使用 DI 容器管理服务生命周期 |
| 代码复用 | 避免重复代码，提取公共逻辑 |
| 异常处理 | 所有接口添加 try-catch 异常捕获 |
| 数据验证 | 前端校验 + 后端 Data Annotations 双重校验 |

## 二、系统整体功能

### 2.1 功能模块总览

系统共包含 **6 大功能模块**：

```
CampusTrade 校园二手交易平台
├── 用户模块      （注册 / 登录 / 个人信息 / 头像上传 / 个人主页）
├── 商品模块      （发布 / 修改 / 删除 / 列表 / 详情）
├── 收藏模块      （收藏 / 取消收藏 / 收藏列表）
├── 搜索模块      （关键词搜索 / 分类筛选 / 模糊搜索）
├── 聊天模块      （发送消息 / 聊天记录）
└── 订单模块      （创建订单 / 查看订单 / 确认完成）
```

### 2.2 用户模块

| 编号 | 功能 | 描述 |
|------|------|------|
| U-01 | 用户注册 | 填写用户名、密码、邮箱等信息完成注册 |
| U-02 | 用户登录 | 用户名 + 密码登录，服务器写入登录 Cookie |
| U-03 | Cookie 认证 | 浏览器自动携带 Cookie 进行身份校验 |
| U-04 | 修改个人信息 | 修改昵称、手机号、邮箱等 |
| U-05 | 上传头像 | 支持图片上传作为用户头像 |
| U-06 | 查看个人主页 | 展示用户基本信息及发布的商品 |

**用户实体字段**：Id、Username、Password（加密）、Email、Avatar、Phone、CreateTime

### 2.3 商品模块

| 编号 | 功能 | 描述 |
|------|------|------|
| P-01 | 发布商品 | 卖家填写标题、描述、价格、分类、图片后发布 |
| P-02 | 修改商品 | 卖家编辑已发布商品的信息 |
| P-03 | 删除商品 | 卖家下架 / 删除已发布商品 |
| P-04 | 商品列表 | 分页展示所有在售商品，支持分类浏览 |
| P-05 | 商品详情 | 查看商品完整信息及卖家信息 |

**商品实体字段**：Id、Title、Description、Price、ImageUrl、Category、Status（出售中 / 已售出）、SellerId、CreateTime

### 2.4 收藏模块

| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 收藏商品 | 用户点击收藏按钮添加商品到收藏列表 |
| F-02 | 取消收藏 | 用户取消已收藏的商品 |
| F-03 | 查看收藏列表 | 在个人中心查看所有已收藏商品 |

### 2.5 搜索模块

| 编号 | 功能 | 描述 |
|------|------|------|
| S-01 | 关键词搜索 | 按商品标题关键字搜索，支持模糊匹配 |
| S-02 | 分类筛选 | 按商品分类进行筛选 |

### 2.6 聊天模块

| 编号 | 功能 | 描述 |
|------|------|------|
| M-01 | 发送消息 | 买卖双方之间发送文字消息 |
| M-02 | 聊天记录 | 查看与指定用户的历史聊天记录 |

**消息实体字段**：Id、SenderId、ReceiverId、Content、SendTime

> 聊天功能第一版使用数据库存储消息，定时轮询刷新，不引入 SignalR 等实时通信框架。

### 2.7 订单模块

| 编号 | 功能 | 描述 |
|------|------|------|
| O-01 | 创建订单 | 买家对商品发起购买，生成订单 |
| O-02 | 查看订单 | 买家查看我的订单 / 卖家查看售出订单 |
| O-03 | 确认完成 | 买家确认收货，订单状态变更为已完成 |

**订单实体字段**：Id、BuyerId、ProductId、Price、Status（待付款 / 交易中 / 已完成）、CreateTime

### 2.8 页面清单

| 序号 | 页面 | 核心内容 |
|------|------|------|
| 1 | 首页 | 搜索栏、分类导航、推荐商品列表 |
| 2 | 登录页 | 用户名 + 密码登录表单 |
| 3 | 注册页 | 用户注册表单 |
| 4 | 商品列表页 | 商品卡片网格、分类筛选、搜索栏 |
| 5 | 商品详情页 | 商品大图、详情信息、卖家信息、收藏/购买按钮 |
| 6 | 发布商品页 | 商品信息表单（标题、描述、价格、分类、图片） |
| 7 | 个人中心页 | 个人信息、头像、我的发布、我的收藏入口 |
| 8 | 订单页 | 订单列表（买家视角/卖家视角）、状态操作按钮 |
| 9 | 聊天页 | 会话列表 + 聊天窗口 |

### 2.9 UI 设计要求
- **风格**：简洁校园风，清爽配色，Bootstrap 组件
- **布局**：响应式布局，适配 PC 及移动端
- **交互**：表单前端校验 + 操作 Toast 反馈提示

## 三、主要业务流程（含交易规则）

### 3.1 用户注册与登录流程

```
用户进入平台 → 点击注册 → 填写信息（用户名、密码、邮箱）→ 提交注册
→ 注册成功 → 跳转登录页 → 输入用户名+密码 → 登录成功
→ 服务器写入登录 Cookie（浏览器自动管理）→ 进入首页（登录态）
```

### 3.2 商品发布流程

```
卖家登录 → 点击"发布商品" → 填写商品信息（标题、描述、价格、分类、图片）
→ 提交 → 前端 fetch 调用 API → 后端校验 → 商品发布成功 → 商品出现在商品列表
```

**交易规则 R1 — 发布权限**：仅登录用户可发布商品。

**交易规则 R2 — 信息完整性**：标题、描述、价格、分类为必填项；图片可选，未上传时使用默认占位图。

### 3.3 商品浏览与搜索流程

```
用户访问首页 → 浏览器请求 API 获取商品列表 → 渲染商品卡片
→ 可继续按关键词搜索 或 切换分类筛选 → 点击商品卡片 → 进入商品详情页
```

**交易规则 R3 — 可见性**：所有商品默认公开可见（包括未登录用户），但仅"出售中"状态的商品在列表中展示。

### 3.4 收藏流程

```
用户在商品详情页 → 点击"收藏" → 前端 fetch 调用 API（浏览器自动携带 Cookie）
→ 收藏成功 → 用户可在个人中心查看收藏列表 → 可随时取消收藏
```

**交易规则 R4 — 收藏限制**：仅登录用户可收藏；同一用户对同一商品不可重复收藏。

### 3.5 聊天流程

```
买家在商品详情页 → 点击"联系卖家" → 进入聊天页 → 发送消息（fetch API）
→ 卖家在聊天列表看到消息 → 进入聊天页回复 → 双方沟通交易细节
```

**交易规则 R5 — 聊天权限**：仅登录用户可发送消息；用户之间可自由发起对话。

### 3.6 下单购买流程

```
买家在商品详情页 → 点击"立即购买" → 前端校验：
  ├── 未登录 → 跳转登录页
  ├── 自己不能买自己的商品
  └── 通过 → fetch API 创建订单（状态：待付款）

买家 → 线下与卖家沟通、验货、付款
卖家 → 确认收款 → 订单状态变更：待付款 → 交易中
买家 → 确认收货 → 订单状态变更：交易中 → 已完成

同时商品状态变更：出售中 → 已售出
```

### 3.7 订单状态流转图

```
                   买家下单
   ┌─────────┐  ───────────>  ┌──────────┐
   │  待付款   │                │  交易中    │
   └─────────┘                └──────────┘
                                    │
                             买家确认收货
                                    │
                                    ▼
                              ┌──────────┐
                              │  已完成    │
                              └──────────┘
```

### 3.8 交易规则汇总

| 规则编号 | 规则内容 |
|------|------|
| R1 | 仅登录用户可发布商品 |
| R2 | 标题、描述、价格、分类为必填项；图片未上传则使用默认占位图 |
| R3 | 仅"出售中"状态的商品在列表中展示；已售出商品不可见 |
| R4 | 仅登录用户可收藏；同一商品不可重复收藏 |
| R5 | 仅登录用户可发送聊天消息 |
| R6 | 用户不能购买自己发布的商品 |
| R7 | 订单生成后，需手动流转状态：待付款 → 交易中 → 已完成 |
| R8 | 订单"已完成"时，对应商品自动标记为"已售出" |
| R9 | 卖家可随时下架"出售中"的商品（删除），已售出商品不可删除 |
